# iOS 雙擊防放大第二階段與 APP 版次識別設計

## 背景與驗證結果

第一階段已部署至 `dev`，手機驗證結果如下：

- 行程卡片非互動文字區（例如「9:00／check in／第一航廈1樓」）快速雙擊後，iOS Dev PWA 仍會原生放大。
- 捏合與回彈正常。
- 桃子徽章雙擊開啟診斷面板正常。
- 輸入框 focus／blur 正常。
- `healthCheck()` 正常。
- 診斷面板顯示 `SW okayama-trip-v4`，證明手機已取得第一階段 App Shell；`SCHEMA 2.1` 未變是正確結果，不代表載入舊版。

目前 `setupDoubleTapGuard()` 僅在第二次 `touchend` 阻止預設行為。手機實測顯示，這個攔截時機在目前的 iOS Dev PWA 環境不足以阻止 WebKit 原生雙擊縮放。

## 目標

1. 阻止非互動文字或空白區的單指快速雙擊縮放。
2. 保留兩指捏合與既有 0.2 秒回彈。
3. 保留桃子徽章雙擊、按鈕、連結、表單及滑動行為。
4. 在診斷面板清楚顯示 APP 環境、功能程式提交碼與日期。
5. 不變更 Schema、Google Sheets、資料解析或個人狀態。

## 採用方案：A — 提前攔截第二次單指觸碰

強化 `setupDoubleTapGuard()`，將攔截判斷提前到第二次 `touchstart`。防護只在下列條件全部成立時呼叫 `preventDefault()`：

- 目前只有一根手指。
- 與前一次候選點擊的間隔不超過 350ms。
- 兩次觸碰位置在小範圍距離門檻內。
- 觸碰位於非互動文字或空白區。
- 前一次觸碰沒有形成滑動、捏合或其他多指手勢。

下列目標不納入一般防護：

- `button`
- `a`
- `input`
- `select`
- `textarea`
- 具有既有互動處理的元素
- `#diagnosticBadge`

桃子徽章繼續使用自己的雙擊處理器。兩指觸碰不進入單指候選狀態，因此 `setupPinchZoom()` 保持不變。

## 替代方案與升級門檻

### B — 雙擊時暫時修改 viewport

不採用。它可能與第一階段的 focus／blur viewport 還原機制互相影響，並帶來畫面跳動或縮放狀態錯亂。

### C — 永久限制所有縮放

本階段不採用，但保留為未來另案評估選項。只有在方案 A 經 Dev 手機重新驗證仍失敗，且重新蒐集 iOS 版本、PWA 啟動環境與觸碰事件證據後，才能提出方案 C。方案 C 必須由 Bar 再次明確核准，因為它會犧牲使用者縮放的可及性，而且 iOS WebKit 可能忽略 `user-scalable=no` 或永久 `maximum-scale=1`，不保證能可靠阻止捏合。

不允許程式在方案 A 失敗時自動切換方案 C。

## APP 版次識別

診斷面板新增獨立一行：

`APP DEV · CODE <功能程式提交碼> · 2026-07-13`

其中：

- `DEV` 表示目前執行環境，不與 Schema 混用。
- `CODE` 指向實際包含本批 App 功能修改的短提交碼。
- 日期代表該功能版次日期。
- `SCHEMA 2.1` 與 Service Worker 版次繼續分別顯示。

Git commit 無法在同一個 commit 內容中正確包含自己的 hash，因此交付流程先建立功能程式提交，再以後續版次識別提交將 `CODE` 指向該功能提交。顯示值代表 App 功能程式版本，不宣稱是分支最新文件提交。

## Service Worker 與離線更新

本批會修改 `index.html`，因此實作時將 Service Worker cache 從 `okayama-trip-v4` 升至 `okayama-trip-v5`，並同步更新診斷面板。既有快取清理及離線 App Shell 行為不變。

## 風險控制

- 快速滑動被誤判：以移動取消條件及距離門檻避免。
- 快速點擊不同區域被誤判：必須同時符合時間與空間門檻。
- 互動元件失效：互動元素與桃子徽章明確排除。
- 捏合失效：任何兩指以上觸碰不進入單指雙擊防護。
- iOS 裝置差異：桌面模擬僅做回歸測試，最後仍以 Bar 的 Dev PWA 手機驗證為準。

## 測試與驗收

自動測試至少覆蓋：

1. 同位置、時限內的第二次單指非互動觸碰會被阻止。
2. 超過時間或距離門檻不會被阻止。
3. 滑動後不形成雙擊候選。
4. 兩指捏合不會被阻止。
5. 按鈕、連結、表單與桃子徽章不會被一般防護攔截。
6. 診斷面板包含 APP、CODE、日期、Schema 與 `SW okayama-trip-v5`。
7. viewport meta 不永久包含 `maximum-scale` 或 `user-scalable=no`。

Dev 手機驗收：

1. 行程卡片的時間、活動及地點文字快速雙擊不放大。
2. 空白區快速雙擊不放大。
3. 一般單擊及上下滑動正常。
4. 捏合與回彈正常。
5. 桃子徽章雙擊正常。
6. 輸入框 focus／blur 正常。
7. `healthCheck()` 正常，診斷面板顯示正確 APP 與 SW 版次。

## 範圍外

- 不修改 `main`。
- 不修改 Schema、Validator、Google Sheets 或同步資料流。
- 不取消捏合縮放。
- 不永久加入 `maximum-scale` 或 `user-scalable=no`。
- 不在本階段執行方案 C。
