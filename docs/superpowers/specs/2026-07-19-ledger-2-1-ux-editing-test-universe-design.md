## 1. 目標

在不改變個人帳本機保存、團體帳 Apps Script append-only 同步、JPY／TWD 雙幣別及四分頁結構的前提下，修正 Ledger 2.0 手機操作問題，讓 TEST 模式可以完整驗證團體結算，並加入可搜尋店名、消費發生時間、代購對象管理、單品免稅、紀錄編輯及進階歷史查詢。

本批同時把團體儲存改成 queue-first 樂觀完成：只要整批資料安全進入本機佇列，立即結束表單操作，網路送達改由背景同步承接。

## 2. 不變事項

- 個人帳只存 `localStorage`，不進 CMS 或團體 Queue。
- 團體帳一律經 Ledger Repository 寫入「分帳紀錄」。
- 團體帳維持 append-only；不得直接改寫或刪除 Sheet 舊列。
- 幣別只支援 JPY／TWD；單筆只輸入一種幣別，另一幣別依共用匯率換算。
- CMS 行前團費邏輯不變。
- 四分頁結構、Scroll-only／viewport recovery、BUILTIN、既有 Apps Script URL 與 ID 去重機制不變。
- 不導入 AbortController、Firebase、Supabase 或新的前端相依套件。

## 3. Schema 2.7 與傳輸契約

### 3.1 `time` 語意

既有 `time` 欄的正式語意由「寫入時間」改為「消費發生時間」。

- 表單預設為裝置目前日期時間。
- 使用者可修改日期與時間。
- UI 使用 `datetime-local`；保存時將裝置本地時間轉為 ISO 8601 字串。
- 今日卡、日期分組、排序與日期篩選全部依 `time`。
- 歷史紀錄不遷移，既有 `time` 值直接視為消費發生時間。
- 本批不新增另一個伺服器寫入時間欄。

### 3.2 Ledger 位置式欄位

既有 14 欄末端追加兩個選填欄位，形成位置式 16 欄契約：

| 位置 | Sheet 表頭 | Property | 規則 |
|---:|---|---|---|
| 1–14 | 既有欄位 | 既有契約 | 欄位順序與語意除 `time` 外不變 |
| 15 | 店名 | `storeName` | 選填、trim 後保存，可搜尋與顯示 |
| 16 | 取代紀錄ID | `replacesRecordId` | 選填；團體編輯的新筆指向被取代紀錄 |

Apps Script 固定 append 16 欄。舊 payload 缺少新欄位時寫入空字串；ID 去重、LockService、`updateSettings` 白名單與回應契約不變。

### 3.3 個人帳模型

個人紀錄同步加入 `storeName` 與 `replacesRecordId`。兩欄只保存在 `trip_personal_ledger`，不得進團體 Queue。個人狀態匯出版本 bump，匯出與還原須保存這兩欄。

## 4. Bottom Sheet 穩定性

### 4.1 開啟與關閉位置

- 開啟新增、明細、結算或編輯 Sheet 時，其內部 scroll surface 從 `scrollTop=0` 開始。
- 開啟前記錄背景 `window.scrollY`。
- 關閉後於下一個 animation frame 還原背景位置。
- 背景仍以既有 `ledger-sheet-open` 鎖定，不新增手勢攔截器或 `preventDefault()`。

### 4.2 選項不得造成跳頂

膠囊、checkbox 與展開控制優先做局部 DOM／class／ARIA 更新，不得重建整個 Sheet。只有欄位結構確實改變時才允許重繪，例如：

- 單品／多品項切換；
- 個人／團體帳切換；
- 代購開關顯示對象區；
- 自訂稅率輸入顯示；
- 單項設定展開。

允許重繪時必須在重繪前保存 Sheet `scrollTop` 與目前 focus，重繪後還原。驗收標準為 Sheet 捲到中段點擊任一膠囊，畫面位置不跳動。

## 5. 新增／編輯表單

### 5.1 欄位順序

1. 標題「新增消費」或「編輯消費」
2. `目前記帳身分：姓名`
3. 多品項開關
4. 帳本：個人／團體
5. 幣別：JPY／TWD
6. 匯率膠囊
7. 金額與行內換算
8. 明細／品項
9. 店名（選填）
10. 消費日期與時間
11. 類別
12. 支付方式
13. 個人代購或團體分攤
14. 稅與優惠（選填、預設收合）
15. 備註
16. 儲存／儲存並再記一筆

### 5.2 金額與匯率

- 金額標籤依幣別顯示 `金額 (¥)` 或 `金額 (NT$)`。
- 換算結果使用 input wrapper 右側的 `<output>` 顯示，例如 `≈ NT$199`；不得把文字直接塞入 `<input>` value。
- 輸入框保留足夠右側 padding，輸入文字不得與換算結果重疊。
- 匯率說明收斂為同區膠囊，例如 `1 JPY ≈ TWD 0.199`。
- 金額輸入維持正整數與安全整數檢查。

### 5.3 店名

- `storeName` 選填。
- 單品與多品項皆為整單層級；同一批品項共用店名。
- 團體帳寫入正式第 15 欄，不再使用備註前綴。
- 個人帳以結構欄位保存。
- 最近紀錄、明細與完整歷史在有值時顯示店名。

### 5.4 類別記憶

- 首筆預設 `餐飲`；若類別清單不含餐飲，使用第一個有效類別。
- 每次成功儲存後保留該類別，作為下一筆預設。
- 多品項新增品項沿用目前整單／上一品項類別。

## 6. 稅與優惠

### 6.1 UI

整區標題為「稅與優惠（選填）」並預設收合。所有主要選項使用既有膠囊樣式：

- 價格方式：`税込（含稅）`、`税抜（未稅）`
- 稅率：`無稅`、`8%`、`10%`、`自訂`
- 自訂被選中時顯示百分比數字輸入。
- 折扣為整單固定金額，幣別跟隨輸入幣別。

自訂稅率必須大於 0、不超過 100%，最多兩位小數；內部轉換為 basis points，避免浮點權重破壞整數守恆。

### 6.2 計算

- `税込（含稅）`：輸入金額即為實付基礎，不再加稅。稅率只用於顯示其中含稅額／免稅可省金額。
- `税抜（未稅）`：可課稅品項依稅率增加金額。
- `無稅`：稅率視為 0%，不增加金額。
- 折扣於計稅後扣除，不得大於計稅後總額，且最終實付必須大於零。
- JPY／TWD 的品項分配沿用最大餘數法，各幣總和必須精確等於整單實付。
- 税抜權重以整數 basis points 計算，整單計稅後總額沿用現行 `Math.round` 最終取整語意。

### 6.3 多品項免稅

- 每個品項新增「免稅品」旗標。
- 免稅品不增加稅，即使整單價格方式為税抜且稅率大於 0。
- 「全部免稅品」按鈕須先確認，再將所有品項旗標設為 true。
- 取消單品免稅只影響該品項。
- 税込模式下旗標不改變實付，只影響免稅可省金額的呈現。

## 7. 代購對象管理

- 新增本機代購對象清單，獨立 localStorage key。
- 表單使用膠囊：`未指定`、已保存對象、`＋新增對象`。
- 新增對象 trim、去空白、去重並套用既有長度上限。
- 設定頁提供新增與刪除；刪除清單項目不修改既有紀錄。
- 多品項的單項代購覆寫使用同一清單。
- 清單納入個人狀態 JSON 匯出、還原、格式驗證與 restore 白名單。

## 8. TEST 平行帳本

TEST 模式只作用於團體帳。

### 8.1 選擇規則

- TEST 開啟：團體儀表板、今日、最近紀錄、完整紀錄、結算與明細只使用 `[TEST]` expense。
- TEST 關閉：上述畫面只使用非 TEST expense。
- 個人帳永遠使用全部個人紀錄，不受 TEST 開關影響。
- 本機待同步紀錄依相同規則進入目前宇宙。
- 有效紀錄解析順序固定為：先合併雲端／Queue／bridge並套用墓碑，再依 TEST 宇宙篩選。
- 正式模式排除 TEST 身分註冊；TEST 模式維持現況，可選正式及 TEST 註冊成員。
- CMS 行前團費不切換。

### 8.2 結算

- 現行結算引擎的「永遠排除 TEST」改為明確 universe 參數。
- 正式結算只接受非 TEST expense；測試結算只接受 TEST expense。
- participants 快照、安全整數、雙幣獨立結算及確定性轉帳規則不變。
- 橫幅改為：`⚠ 目前顯示測試帳本`／`不影響正式分帳`，並可前往設定關閉。

## 9. 樂觀儲存與非同步狀態

### 9.1 Repository 邊界

保留 `ledgerRepository.add()` 既有「嘗試送達後完成」語意，新增固定批次介面 `enqueueBatch(records)`：

- 先完整正規化及驗證整批紀錄。
- 一次性寫入 localStorage Queue；寫入失敗時不留下半批資料。
- 成功後立即回傳 queued acknowledgment。
- 背景呼叫既有 `flushQueue()`；POST 結果不阻塞表單。
- 伺服器 `{ok:true}`／`{ok:true,dup:true}` 才由既有流程移出 Queue。

Repository 新介面及完成語意須補入 ADR 0006。

### 9.2 UI

- 團體帳：整批入列成功即關閉 Sheet，toast `已儲存，待同步`。
- `儲存並再記一筆`：整批入列成功即重設 draft，保留幣別、類別、支付方式及團體分攤成員。
- 背景失敗保留 Queue，由「N 筆待同步」及連網／開 App 補送承接。
- 個人帳在 localStorage 原子寫入成功後立即完成。
- 設定儲存與手動同步等待網路時顯示 spinner＋disabled。
- 樂觀記帳不顯示長時間 spinner；只在本機入列極短期間 disabled，避免重複點擊。

## 10. 編輯與刪除

### 10.1 個人帳

- 單筆編輯直接替換 localStorage 原紀錄並保留 ID。
- 多品項以 batch 為單位原子替換；既有對應品項盡量保留 ID，新增品項產生新 ID，移除品項直接從本機 batch 移除。
- 既有雙重確認真刪規則不變。

### 10.2 團體帳

團體帳不得修改 Sheet 舊列。編輯流程固定為：

1. 載入目前有效的單筆或完整 batch。
2. 建立每筆舊紀錄的 deletion 墓碑，`deleteReason='編輯修改'`。
3. 以修改後內容建立新紀錄／新 batch。
4. 新單筆的 `replacesRecordId` 指向舊筆 ID。
5. 新 batch 每筆的 `replacesRecordId` 指向舊 batch 依時間、ID 排序後的第一筆 ID，作為本次取代根。
6. 所有墓碑與新筆一次性進入 Queue，避免半套編輯。

正式紀錄編輯後仍為正式；TEST 紀錄編輯後仍具有 `[TEST]` 前綴。是否為 TEST 由原紀錄決定，不讀取編輯當下的模式開關。

若原紀錄仍在本機 Queue 或送達中，仍使用相同墓碑＋新筆模型，並保證 Queue 順序為舊筆、墓碑、新筆；不得直接竄改可能已送達的 payload。

團體刪除維持既有必填原因；個人刪除維持既有雙重確認。

## 11. 完整紀錄

擴充既有完整紀錄頁，不建立新的第五分頁：

- 搜尋：明細／品項名稱、店名、備註。
- 篩選膠囊：類別、支付方式、代購。
- 分組切換：按日期／按類別。
- TEST 不提供獨立 filter；目前模式已決定正式或測試宇宙。
- 所有搜尋與篩選只影響顯示，不修改資料。
- 編輯／刪除由每筆 `⋯` 選單進入。
- 團體墓碑、已刪目標與不可操作的系統紀錄不顯示操作選單。

## 12. 相容性與錯誤處理

- 舊 14 欄 CSV 的新欄位解析為空字串。
- 舊個人紀錄的 `storeName`、`replacesRecordId` 預設為空字串。
- 舊 `time` 不重寫，直接以新語意呈現。
- 無效消費日期、店名、稅率、折扣、代購對象或整批資料一律在進 Queue 前阻擋。
- localStorage 寫入失敗不得關閉 Sheet，並保留完整 draft。
- 背景 POST 失敗不得撤銷本機已儲存紀錄。
- CSV 發布延遲仍為 1–5 分鐘的已知架構取捨。
- 搜尋與解析遇到舊紀錄缺欄時安全降級，不得造成 pageerror。
